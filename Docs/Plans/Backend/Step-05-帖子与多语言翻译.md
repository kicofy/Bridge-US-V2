# Step 05 帖子与多语言翻译

## 目标
实现帖子完整生命周期，并在发布时生成多语言版本保存。

## 前置条件
- Step 04 完成

## 详细任务拆解
1. 帖子基础接口
   - 创建 / 查询 / 更新 / 删除
   - 分页与排序（最新/有用/准确）
   - 状态管理：draft/published/hidden
2. 发布流程
   - `posts` 写入原文
   - 触发翻译任务（队列或同步）
   - 写入 `post_translations`
3. 多语言读取
   - 根据 `language` 返回对应版本
   - 无翻译回退原文
   - 记录翻译状态（pending/ready/failed）
4. 分类与标签
   - 绑定分类与标签
   - 校验分类有效性
5. AI 内容筛查
   - 发布前广告/推广检测
   - 不安全内容进入审核

## 产出物
- `/posts/*` 接口完整
- `post_translations` 写入逻辑
- 多语言读写流程

## 验收标准
- 发布后自动生成多语言记录
- 同一帖子多语言可正确读取
- 违规内容进入审核流程

## 风险与注意
- 翻译失败需可重试
- 多语言内容需可追踪来源（AI/人工）
